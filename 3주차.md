# 1️⃣ 개념 정리 

## 01. 불필요한 데이터 삭제하기

### 📌 데이터 정제란?
수집된 데이터에서 손상되거나 부정확한 부분을 수정하고, 불필요한 데이터를 삭제하거나 불완전한 값을 교체하는 작업을 **데이터 정제(data cleaning)**라고 한다. 이는 데이터를 분석 목적에 맞게 변환하는 **데이터 랭글링(data wrangling)** 또는 **데이터 먼징(data munging)**의 일부로 수행된다.

---

### 📌 열 삭제하기

**방법 1. loc 메서드 + 슬라이싱**
```python
ns_book = ns_df.loc[:, '번호':'등록일자']  # 특정 범위 열 선택
```

**방법 2. loc 메서드 + 불리언 배열**
```python
selected_columns = ns_df.columns != 'Unnamed: 13'
ns_book = ns_df.loc[:, selected_columns]
```
> 판다스의 Index 클래스를 어떤 값과 비교하면 원소 하나하나와 비교하여 True/False 배열을 반환하는데 이를 **원소별 비교(element-wise comparison)**라 한다.

**방법 3. drop() 메서드**
```python
ns_book = ns_df.drop('Unnamed: 13', axis=1)          # 열 1개 삭제
ns_book = ns_df.drop(['부가기호', 'Unnamed: 13'], axis=1)  # 열 여러 개 삭제
ns_book.drop('주제분류번호', axis=1, inplace=True)    # 원본 직접 수정
```
> `axis=1`이면 열 삭제, `axis=0`(기본값)이면 행 삭제

**방법 4. dropna() 메서드**
```python
ns_book = ns_df.dropna(axis=1)           # NaN이 하나라도 있는 열 삭제
ns_book = ns_df.dropna(axis=1, how='all')  # 모든 값이 NaN인 열만 삭제
```

---

### 📌 행 삭제하기

**방법 1. drop() 메서드**
```python
ns_book2 = ns_book.drop([0, 1])  # 인덱스 0, 1 행 삭제
```

**방법 2. [] 연산자 + 슬라이싱**
```python
ns_book2 = ns_book[2:]    # 인덱스 2부터 끝까지
ns_book2 = ns_book[0:2]   # 인덱스 0, 1만 선택 (2는 미포함)
```
> `loc` 메서드의 슬라이싱은 마지막 인덱스를 **포함**하지만, `[]` 연산자의 슬라이싱은 마지막 인덱스를 **포함하지 않는다.**

**방법 3. [] 연산자 + 불리언 배열 (가장 자주 사용)**
```python
selected_rows = ns_df['출판사'] == '한빛미디어'
ns_book2 = ns_book[selected_rows]

# 조건을 직접 넣는 방식
ns_book2 = ns_book[ns_book['대출건수'] > 1000]
```

---

### 📌 중복된 행 찾기: duplicated()
```python
sum(ns_book.duplicated())  # 중복 행 개수 확인

# 특정 열 기준으로 중복 탐색
sum(ns_book.duplicated(subset=['도서명', '저자', 'ISBN']))

# 중복된 모든 행을 True로 표시
dup_rows = ns_book.duplicated(subset=['도서명', '저자', 'ISBN'], keep=False)
```
> 기본적으로 처음 등장한 행은 False, 이후 중복된 행은 True로 표시한다.  
> `keep=False`로 지정하면 중복된 **모든** 행을 True로 표시한다.

---

### 📌 그룹별 모으기: groupby() + sum()
같은 도서가 여러 권 등록된 경우 대출건수를 하나로 합산할 때 사용한다.
```python
loan_count = count_df.groupby(
    by=['도서명', '저자', 'ISBN', '권'],
    dropna=False
).sum()
```
> `dropna=False`를 지정하면 NaN이 있는 행도 계산에 포함된다.

---

### 📌 원본 데이터 업데이트하기
```python
# ① 중복 제거 후 고유 행만 추출
dup_rows = ns_book.duplicated(subset=['도서명', '저자', 'ISBN', '권'])
unique_rows = ~dup_rows         # ~ 연산자로 불리언 배열 반전
ns_book3 = ns_book[unique_rows].copy()

# ② 인덱스 설정
ns_book3.set_index(['도서명', '저자', 'ISBN', '권'], inplace=True)

# ③ 대출건수 업데이트
ns_book3.update(loan_count)

# ④ 인덱스 해제
ns_book4 = ns_book3.reset_index()

# ⑤ 열 순서 원복
ns_book4 = ns_book4[ns_book.columns]
```

### 📌 핵심 함수 정리

| 함수/메서드 | 기능 |
|------------|------|
| `drop()` | 행이나 열 삭제 |
| `dropna()` | NaN 포함 행/열 삭제 |
| `duplicated()` | 중복된 행 탐색 (불리언 배열 반환) |
| `drop_duplicates()` | 중복된 행 삭제 |
| `groupby()` | 특정 열 기준으로 행 그룹화 |
| `sum()` | 합계 계산 |
| `set_index()` | 열을 인덱스로 설정 |
| `reset_index()` | 인덱스 재설정 |
| `update()` | 다른 데이터프레임으로 값 업데이트 |
| `equals()` | 두 데이터프레임이 동일한지 비교 |

## 02. 잘못된 데이터 수정하기

### 핵심 키워드
- **NaN**: 판다스에서 누락된 값을 표시하는 기호
- **정규 표현식**: 문자열에서 패턴을 찾고 대체하기 위한 규칙의 모음

---

### 데이터프레임 정보 요약 확인하기

`info()` 메서드를 사용하면 열마다 NaN이 아닌 값이 몇 개인지 확인할 수 있다.
```python
ns_book4.info()
```

출력 항목:
- 전체 행 개수 / 열 개수 / 열 이름
- 각 열의 Non-Null Count (누락되지 않은 값의 수)
- 각 열의 데이터 타입 (float64, int64, object 등)
- 전체 메모리 사용량

> 정확한 메모리 사용량을 확인하려면 `info(memory_usage='deep')` 옵션을 사용한다.

---

### 누락된 값 처리하기

#### 누락된 값 개수 확인: `isna()` + `sum()`

`isna()` 메서드는 각 행이 비어 있는지를 나타내는 **불리언 배열**을 반환한다.  
`sum()`을 이어서 호출하면 True의 개수, 즉 누락된 값의 수를 얻을 수 있다.
```python
ns_book4.isna().sum()
```

반대로 누락되지 않은 값을 확인할 때는 `notna()` 메서드를 사용한다.

---

#### 누락된 값으로 표시하기: `None`과 `np.nan`

- 정수형 열에 `None`을 입력하면 NaN으로 인식되며, 해당 열의 타입이 **float64로 자동 변환**된다.
- 문자열(object) 열에 `None`을 입력하면 NaN이 아닌 **문자열 그대로 None**으로 표시된다.
- 데이터 타입에 상관없이 NaN으로 표시하려면 **넘파이의 `np.nan`** 을 사용해야 한다.
```python
import numpy as np
ns_book4.loc[0, '부가기호'] = np.nan
```

열의 데이터 타입을 변환할 때는 `astype()` 메서드를 사용하며, 딕셔너리 형식으로 전달한다.
```python
ns_book4 = ns_book4.astype({'도서권수': 'int32', '대출건수': 'int32'})
```

---

#### 누락된 값 바꾸기 (1): `loc` + `fillna()` 메서드

**loc 사용법**: `isna()`로 불리언 배열을 만들어 특정 위치의 NaN을 원하는 값으로 교체
```python
set_isbn_na_rows = ns_book4['세트 ISBN'].isna()
ns_book4.loc[set_isbn_na_rows, '세트 ISBN'] = ''
```

**fillna() 사용법**: 원하는 값을 전달하면 NaN을 일괄 대체 (새로운 데이터프레임 반환)
```python
ns_book4.fillna('없음')                          # 전체 NaN 대체
ns_book4['부가기호'].fillna('없음')              # 특정 열만 대체
ns_book4.fillna({'부가기호': '없음'})            # 딕셔너리로 열 지정
```

---

#### 누락된 값 바꾸기 (2): `replace()` 메서드

NaN뿐만 아니라 **어떤 값도 바꿀 수 있는** 메서드.

| 사용 방식 | 코드 예시 |
|---|---|
| 값이 하나일 때 | `replace(원래값, 새로운값)` |
| 값이 여러 개일 때 | `replace([원래1, 원래2], [새로운1, 새로운2])` |
| 딕셔너리로 전달 | `replace({원래1: 새로운1, 원래2: 새로운2})` |
| 열마다 다른 값 | `replace({열이름: 원래값}, 새로운값)` |
| 열마다 다르게 (중첩) | `replace({열이름: {원래값: 새로운값}})` |
```python
ns_book4.replace(np.nan, '없음')
ns_book4.replace([np.nan, '2021'], ['없음', '21'])
ns_book4.replace({'부가기호': np.nan}, '없음')
ns_book4.replace({'부가기호': {np.nan: '없음'}, '발행년도': {'2021': '21'}})
```

---

### 정규 표현식

**정규 표현식(regular expression)** 은 문자열 패턴을 찾아서 대체하기 위한 규칙의 모음이다.  
`replace()` 메서드에 `regex=True`를 지정하면 정규 표현식을 사용할 수 있다.

#### 주요 기호

| 기호 | 의미 |
|---|---|
| `\d` | 숫자 한 자리 |
| `\D` | 숫자가 아닌 문자 |
| `\s` | 공백 문자 |
| `.` | 어떤 문자 한 자리 |
| `*` | 0개 이상 반복 |
| `{n}` | n번 반복 (예: `\d{4}` = 숫자 4개) |
| `( )` | 그룹 지정 |
| `\1`, `\2` | 첫 번째, 두 번째 그룹 참조 |
| `\(` | 괄호를 일반 문자로 인식 |

> 정규 표현식 문자열 앞에는 `r`을 붙여 일반 문자열과 구분한다. (예: `r'\d{4}'`)

#### 활용 예시
```python
# 발행년도에서 숫자 4자리만 추출
ns_book4.replace({'발행년도': {r'\d\d(\d\d)': r'\1'}}, regex=True)
ns_book4.replace({'발행년도': {r'\d{2}(\d{2})': r'\1'}}, regex=True)  # 동일한 표현

# 저자 열에서 '(지은이)', '(옮긴이)' 제거
ns_book4.replace(
    {'저자': {r'(.*)\s\(지은이\)(.*)\s\(옮긴이\)': r'\1\2'}},
    regex=True
)

# 연도 앞뒤 불필요한 문자 제거 (예: '1988.', '[2019]')
ns_book5 = ns_book4.replace({'발행년도': r'.*(\d{4}).*'}, r'\1', regex=True)
```

### 잘못된 값 바꾸기

#### 문자열 패턴 검사: `str.contains()`

시리즈에서 문자열 패턴을 포함하는지 검사하며, 기본적으로 정규 표현식을 인식한다.
```python
# 숫자가 아닌 문자가 포함된 행 찾기 (누락값도 True로 처리)
invalid_number = ns_book4['발행년도'].str.contains('\D', na=True)
```

> `== '1988'`은 정확히 일치하는 값만 찾지만, `contains('1988')`은 포함된 모든 행을 찾는다.

#### 비교 메서드

| 메서드 | 부등호 | 내용 |
|---|---|---|
| `gt()` | `>` | 지정된 값보다 큰 값 |
| `ge()` | `>=` | 크거나 같은 값 |
| `lt()` | `<` | 작은 값 |
| `le()` | `<=` | 작거나 같은 값 |
| `eq()` | `==` | 같은 값 |
| `ne()` | `!=` | 같지 않은 값 |

결과 df3를 보면 df1의 col1과 df2의 col3를 기준으로 병합하고 있으며, 두 데이터프레임 중 어느 한쪽에만 있는 값도 모두 포함되어 NaN으로 채워진 것을 확인할 수 있다. 
이는 how='outer' （외부 조인(outer join)）의 결과다. left_on='col1', right_on='col3'으로 기준 열을 지정해야 하며, how='outer'를 사용해야 양쪽 모두의 행이 보존된다. 
how='inner'라면 col1과 col3에 공통된 값(x, y)만 남아 z와 w 행이 사라진다。
